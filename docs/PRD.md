# PRD - 搜尋食譜系統

## 1. 專案概述
- **背景與動機**：現代人常會忘記好吃的食譜在哪裡，或是找不到當下想做的料理。為了解決這個問題，希望能有一個專屬系統，讓人可以輕鬆找到想要的食譜並儲存起來。
- **目標用戶**：我（個人使用者）
- **核心價值主張**：提供一個輕量、簡單且具備推薦機制的食譜搜尋與管理平台，讓使用者能快速找到並收藏理想的料理作法。

## 2. 功能需求
1. **搜尋食譜**
   - 作為使用者，我希望可以透過輸入關鍵字來搜尋食譜，以便快速找到我想做的料理。
2. **儲存食譜**
   - 作為使用者，我希望可以將找到或喜歡的食譜儲存到個人的收藏夾中，以便未來隨時查看。
3. **推薦食譜**
   - 作為使用者，我希望系統可以根據我的喜好或隨機推薦食譜，以便在我不知道要吃什麼時獲得靈感。
4. **瀏覽食譜**
   - 作為使用者，我希望可以瀏覽所有已儲存的食譜列表與詳細作法，以便在做菜時能照著步驟進行。
5. **分類與標籤管理**
   - 作為使用者，我希望可以對儲存的食譜加上分類或標籤（如：中式、西式、甜點），以便更有系統地整理我的食譜庫。

## 3. 非功能需求
- **技術限制**：使用 Flask 作為後端框架，Jinja2 負責渲染前端頁面，並搭配 SQLite 作為輕量級資料庫。
- **效能與安全考量**：
  - 系統搜尋反應時間應在 1 秒內，提供流暢的使用體驗。
  - 基本的防護措施（如防止 SQL Injection 及 XSS 攻擊），確保系統與資料安全。

## 4. MVP 範圍
- **Must Have（必備功能）**：
  - 搜尋食譜（以關鍵字查找食譜名稱或材料）
  - 儲存與瀏覽食譜（新增至個人收藏並能查看詳細內容）
- **Should Have（應有功能）**：
  - 推薦食譜（隨機顯示或依據既有收藏進行簡單推薦）
- **Nice to Have（加分功能）**：
  - 分類與標籤管理
