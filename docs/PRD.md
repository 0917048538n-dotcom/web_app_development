# 搜尋食譜系統 - 產品需求文件 (PRD)

## 1. 專案概述
- **背景與動機**：對於想要做料理的人來說，常常面臨不知道該煮什麼、或者找到食譜卻忘記儲存的困擾。本系統希望能提供一個專屬的平台，讓料理愛好者能夠輕鬆尋找靈感、記錄並分享自己的食譜。
- **目標用戶**：想要做料理的人、日常開伙的家庭主婦/夫、對烹飪有興趣的初學者。
- **核心價值主張**：提供一個直覺、方便的食譜搜尋與管理平台，讓使用者能快速找到適合的料理步驟，並建立專屬的食譜庫。

## 2. 功能需求
以下為系統的核心功能與對應的使用者故事：

1. **查找食譜**
   - 作為一名做料理的人，我希望能夠透過關鍵字或分類搜尋食譜，以便快速找到我想做的料理。
2. **儲存食譜**
   - 作為一名做料理的人，我希望能夠將喜歡的食譜收藏起來，以便未來要煮的時候可以快速查看。
3. **撰寫食譜**
   - 作為一名做料理的人，我希望能夠建立並發布自己的私房食譜，以便記錄料理步驟並分享給其他人。
4. **評分與評論食譜**
   - 作為一名做料理的人，我希望能夠對嘗試過的食譜留下評分和評論，以便提供其他使用者參考，並記錄自己的心得。
5. **產生購物清單**
   - 作為一名做料理的人，我希望能夠從食譜中直接將所需食材加入購物清單，以便我去超市採買時不會遺漏食材。

## 3. 非功能需求
- **技術限制**：使用 Python (Flask), Jinja2 模板引擎與 SQLite 資料庫進行開發。
- **效能考量**：系統回應時間需在合理範圍內，提供流暢的瀏覽與搜尋體驗。
- **安全考量**：使用者密碼需加密儲存，且具備基本的防 SQL Injection 與 XSS 攻擊防護，確保使用者資料安全。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**：
  - 使用者註冊與登入
  - 查找食譜功能 (基本關鍵字搜尋)
  - 檢視食譜詳細內容 (包含食材清單與料理步驟)
  - 撰寫並發布基本食譜
- **Should Have (應該有)**：
  - 儲存/收藏食譜功能
  - 食譜分類標籤 (如：中式、西式、甜點、素食等)
- **Nice to Have (可以有)**：
  - 評分與評論食譜
  - 從食譜一鍵產生購物清單
  - 食譜成品與步驟的圖片上傳功能
