# PRD - 食譜收藏夾系統

## 1. 專案概述
- **背景與動機**：在網路上或生活中看到不錯的食譜，或是自己研發了新的菜色，往往缺乏一個個人專屬的集中平台來記錄與整理。導致日後想做同樣的菜時難以尋找，或是想與他人分享時不夠方便。
- **目標用戶**：個人使用者（以「我」為主），以及未來潛在的親友群體（作為被分享食譜的對象）。
- **核心價值主張**：提供一個直覺、輕量且專屬的食譜管理空間，讓使用者能夠輕鬆新增食譜、記錄詳細材料與作法，並能透過搜尋快速找到所需食譜，甚至分享給他人。

## 2. 功能需求
1. **新增食譜**
   - 作為使用者，我希望能夠新增食譜的基本資訊（如名稱、簡介、圖片連結），以便把我想記錄的菜色存入系統。
2. **列出材料與步驟**
   - 作為使用者，我希望在查看單一食譜時，能清楚看到結構化的所需材料清單與詳細製作步驟，以便做菜時能照表操課。
3. **搜尋食譜**
   - 作為使用者，我希望能夠透過關鍵字搜尋食譜名稱或材料，以便在眾多收藏中快速找到我想做的特定菜色。
4. **編輯與刪除食譜**
   - 作為使用者，我希望能夠隨時修改或刪除既有的食譜內容，以便在食譜有改良或不再需要時保持系統資料更新。
5. **瀏覽食譜列表**
   - 作為使用者，我希望在首頁就能瀏覽所有已儲存的食譜清單，並有簡單的預覽，以便我能快速瀏覽並挑選料理靈感。

## 3. 非功能需求
- **技術限制**：預設使用 Python Flask + Jinja2 + SQLite 作為核心 Web 應用程式與資料庫技術棧。
- **效能與安全考量**：
  - 系統頁面載入與搜尋回饋需在 1 秒內完成，確保良好的個人使用體驗。
  - 需要防範基本的 SQL Injection 及 XSS 攻擊，確保食譜文字與輸入欄位不會破壞系統或頁面顯示。

## 4. MVP 範圍
- **Must Have（必備功能）**：
  - 新增食譜（包含標題、簡介、材料、步驟）
  - 在食譜的細節頁面列出材料與步驟
  - 瀏覽所有收藏的食譜列表
- **Should Have（應有功能）**：
  - 關鍵字搜尋食譜功能（依標題或內容）
  - 編輯與刪除食譜的功能
- **Nice to Have（加分功能）**：
  - 產出食譜的獨立連結以方便「告訴他人」（分享功能）
  - 食譜分類/標籤功能
